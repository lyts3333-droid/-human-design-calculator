# 測試指南

## 🚀 快速測試步驟

### 步驟 1：啟動服務器

#### 方法 1：使用批處理文件
雙擊運行 `start_server.bat` 或 `完整安裝.bat`

#### 方法 2：命令行
```bash
python app.py
```

看到以下訊息表示啟動成功：
```
人類圖計算器 API 伺服器
============================================================
伺服器正在啟動...
訪問地址: http://localhost:5000
```

### 步驟 2：打開瀏覽器測試

1. **訪問應用**
   - 打開瀏覽器
   - 訪問：`http://localhost:5000`

2. **測試註冊功能**
   - 應該看到登錄界面
   - 點擊"立即註冊"
   - 輸入：
     - 用戶名：`test`（至少3個字符）
     - 密碼：`test123`（至少6個字符）
     - 郵箱：（選填）
   - 點擊"註冊"
   - ✅ 應該自動登錄並顯示主應用界面

3. **測試登出功能**
   - 點擊右上角的"登出"按鈕
   - ✅ 應該返回登錄界面

4. **測試登錄功能**
   - 輸入剛才註冊的用戶名和密碼
   - 點擊"登錄"
   - ✅ 應該成功登錄並顯示主應用界面

5. **測試計算功能**
   - 填寫出生資料：
     - 年份：2000
     - 月份：1
     - 日期：1
     - 時間：12:00
     - 姓名：測試
     - 選擇地區、縣市、區域
   - 點擊"計算人類圖"
   - ✅ 應該顯示計算結果

6. **測試歷史記錄保存**
   - 計算完成後，查看"歷史記錄"區域
   - ✅ 應該看到剛才的計算記錄
   - 點擊"載入"按鈕
   - ✅ 應該自動填充表單並顯示結果

7. **測試重新登錄後記錄保留**
   - 點擊"登出"
   - 重新登錄
   - ✅ 歷史記錄應該還在

## 🔍 詳細測試項目

### 測試 1：數據庫初始化
**檢查點：**
- 運行服務器後，檢查項目文件夾中是否生成了 `human_design.db` 文件
- ✅ 如果文件存在，表示數據庫初始化成功

### 測試 2：用戶註冊
**測試用例：**
1. 用戶名太短（少於3個字符）
   - 預期：顯示錯誤訊息
2. 密碼太短（少於6個字符）
   - 預期：顯示錯誤訊息
3. 用戶名已存在
   - 預期：顯示"用戶名已存在"錯誤
4. 正常註冊
   - 預期：註冊成功並自動登錄

### 測試 3：用戶登錄
**測試用例：**
1. 錯誤的用戶名或密碼
   - 預期：顯示"用戶名或密碼錯誤"
2. 正確的用戶名和密碼
   - 預期：登錄成功

### 測試 4：數據保存
**測試步驟：**
1. 登錄後進行一次計算
2. 檢查瀏覽器開發者工具（F12）的 Network 標籤
3. 查看是否有 `POST /api/history` 請求
4. ✅ 如果狀態碼是 201，表示保存成功

### 測試 5：數據加載
**測試步驟：**
1. 創建多條計算記錄（至少2條）
2. 登出並重新登錄
3. 檢查歷史記錄列表
4. ✅ 應該看到所有之前的記錄

### 測試 6：數據刪除
**測試步驟：**
1. 在歷史記錄中點擊"刪除"按鈕
2. ✅ 記錄應該從列表中消失
3. 重新登錄
4. ✅ 刪除的記錄不應該再出現

## 🛠️ 使用瀏覽器開發者工具測試 API

### 打開開發者工具
按 `F12` 或右鍵選擇"檢查"

### 測試 API 端點

#### 1. 測試註冊 API
```javascript
fetch('/api/register', {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    credentials: 'include',
    body: JSON.stringify({
        username: 'testuser',
        password: 'test123',
        email: 'test@example.com'
    })
})
.then(r => r.json())
.then(console.log)
```

#### 2. 測試登錄 API
```javascript
fetch('/api/login', {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    credentials: 'include',
    body: JSON.stringify({
        username: 'testuser',
        password: 'test123',
        remember: true
    })
})
.then(r => r.json())
.then(console.log)
```

#### 3. 測試獲取用戶信息
```javascript
fetch('/api/user', {
    method: 'GET',
    credentials: 'include'
})
.then(r => r.json())
.then(console.log)
```

#### 4. 測試獲取歷史記錄
```javascript
fetch('/api/history', {
    method: 'GET',
    credentials: 'include'
})
.then(r => r.json())
.then(console.log)
```

## 🐛 常見問題排查

### 問題 1：無法訪問網站
**檢查：**
- 服務器是否正在運行
- 端口 5000 是否被占用
- 防火牆是否阻止連接

### 問題 2：註冊/登錄失敗
**檢查：**
- 瀏覽器開發者工具（F12）的 Console 標籤是否有錯誤
- Network 標籤查看 API 請求是否成功
- 服務器終端是否有錯誤訊息

### 問題 3：歷史記錄不顯示
**檢查：**
- 是否已登錄
- 瀏覽器開發者工具的 Network 標籤
- 查看 `/api/history` 請求的響應

### 問題 4：數據庫文件未創建
**檢查：**
- 服務器是否正常啟動
- 是否有寫入權限
- 查看服務器終端的錯誤訊息
