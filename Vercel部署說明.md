# Vercel 部署說明

## ⚠️ 重要提示

Vercel 的 Serverless Functions **不支持文件系統數據庫**（如 SQLite），因此：

1. **登錄/註冊功能在 Vercel 上不可用**
   - 用戶註冊和登錄 API 會返回 503 錯誤
   - 前端會自動回退到 localStorage 模式

2. **歷史記錄功能在 Vercel 上使用 localStorage**
   - 數據保存在瀏覽器的 localStorage 中
   - 重新打開瀏覽器時，記錄仍然存在
   - 但清除瀏覽器數據會丟失記錄

3. **人類圖計算功能完全正常**
   - 所有計算功能都可以正常使用
   - 不受 Vercel 環境限制

## 🔧 已實現的修復

### 後端修復（app.py）
- 自動檢測 Vercel 環境
- 在 Vercel 上使用內存數據庫（不持久化）
- 登錄/註冊 API 返回 503 錯誤（提示使用本地部署）
- 歷史記錄 API 返回 503 錯誤（前端自動回退）

### 前端修復（index.html）
- 自動檢測 API 是否可用
- 如果 API 返回 503，自動回退到 localStorage
- 無縫切換，用戶無感知

## 📋 部署步驟

1. **確保已上傳所有文件到 GitHub**
   - 包括 `vercel.json` 配置文件

2. **在 Vercel 中連接 GitHub 倉庫**
   - 登錄 Vercel
   - 點擊 "New Project"
   - 選擇你的 GitHub 倉庫

3. **Vercel 會自動檢測配置**
   - 使用 `vercel.json` 中的配置
   - 自動部署

4. **環境變量（可選）**
   - 如果需要，可以在 Vercel 設置中添加：
     - `SECRET_KEY`: Flask 的密鑰（用於會話）

|------|---------|------------|
| 人類圖計算 | ✅ | ✅ |

## 🚀 推薦方案

### 方案 1：使用 Vercel（簡單，免費）
- ✅ 適合：展示和測試
- ⚠️ 限制：無登錄功能，歷史記錄僅在瀏覽器中

### 方案 2：使用 Render（完整功能）
- ✅ 適合：需要完整功能
- ✅ 優點：支持文件系統，所有功能可用
- 📝 需要：創建 `render.yaml` 配置文件

### 方案 3：使用外部數據庫（最佳）
- ✅ 適合：生產環境
- ✅ 優點：數據持久化，支持多用戶
- 📝 需要：配置 PostgreSQL 或其他數據庫服務

## 🔍 故障排查

### 問題：仍然看到 500 錯誤
**解決方法：**
1. 檢查 Vercel 部署日誌
2. 確認 `vercel.json` 已上傳
3. 確認所有依賴都在 `requirements.txt` 中

### 問題：登錄界面一直顯示
**解決方法：**
- 這是正常的，因為 Vercel 不支持登錄功能
- 前端會自動檢測並跳過登錄（可能需要刷新頁面）

### 問題：歷史記錄不顯示
**解決方法：**
- 檢查瀏覽器是否允許 localStorage
- 打開開發者工具（F12）查看 Console 錯誤

## 📝 注意事項

1. **首次部署可能需要幾分鐘**
2. **如果修改了代碼，需要重新推送到 GitHub**
3. **Vercel 會自動重新部署**
